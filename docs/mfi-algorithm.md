# MFI（Mail Freshness Index）アルゴリズム解説

## 概要

MFI（メール鮮度指数）は、メールボックスの「健康状態」を **0〜100 のスコア** で表す指標です。

- **100** ：未読ゼロ、または全て直近に届いたメール（完璧な状態）
- **0 に近い**：長期間放置された未読が大量にある（危険な状態）

「未読が多いこと」ではなく、**「古い未読が多いこと」** を問題として扱います。  
今届いたメールはほぼペナルティなし。1週間放置したメールは大きなペナルティです。

---

## Step 1 ： Unread Debt（未読負債）の計算

未読メールを「年齢」に応じた重みで加算します。

### 重みテーブル

| 未読の経過時間 | Weight |
|:------------|-------:|
| 1時間以内     | × 0.2  |
| 1時間〜24時間  | × 1.0  |
| 1日〜3日      | × 3.0  |
| 3日超        | × 8.0  |

### 計算式

```
UnreadDebt = 0.2 × (1時間以内の未読数)
           + 1.0 × (1〜24時間の未読数)
           + 3.0 × (1〜3日の未読数)
           + 8.0 × (3日超の未読数)
```

### 計算例

| 状況                     | Debt  |
|:------------------------|------:|
| 今届いたメール 5件          |   1.0 |
| 昨日のメール 5件            |   5.0 |
| 3日前のメール 5件           |  15.0 |
| 1週間前のメール 5件          |  40.0 |

→ 「1週間放置した5件」は「今届いた5件」の **40倍** の負債になります。

---

## Step 2 ： Personal Baseline（個人基準）の計算

メール量は人によって大きく異なるため、固定の基準は使いません。  
**自分自身の過去30日間の Debt の中央値**を基準とします。

```
PersonalBaseline = 過去30日の UnreadDebt スナップショットの中央値
```

> **なぜ中央値？**  
> 平均値は「たまたま多忙だった1日」で大きく歪みます。  
> 中央値は外れ値に強く、「その人の普段の状態」をより正確に反映します。

### ブートストラップ（データが少ない初期）

スナップショットが3件未満の段階では、過去データが不十分なため、  
現在の Debt の2倍を仮の Baseline として使用します。

```
Baseline（初期）= max(1, UnreadDebt × 2)
```

これにより、初回から **MFI ≈ 60** の現実的なスコアでスタートします。  
（履歴が溜まるにつれて、自分の実績ベースに切り替わります）

---

## Step 3 ： MFI の計算

```
MFI = 100 × exp( −UnreadDebt / PersonalBaseline )
```

指数関数を使う理由：

- Debt がゼロ → `exp(0)` = 1 → MFI = **100**（完璧）
- Debt = Baseline → `exp(−1)` ≈ 0.368 → MFI ≈ **36.8**（平均と同じ状態）
- Debt = Baseline の2倍 → `exp(−2)` ≈ 0.135 → MFI ≈ **13.5**（深刻）

「普段より良い」とスコアが上がりやすく、「普段より悪い」と急激に下がる  
非線形な特性が、行動改善のモチベーションを高めます。

### MFI スコアの目安

| MFI       | 状態         |
|:----------|:------------|
| 80 〜 100  | 優良         |
| 60 〜 79   | 良好         |
| 40 〜 59   | 要注意       |
| 0 〜 39    | 危険（DM通知） |

---

## Step 4 ： Streak Bonus（継続ボーナス）

Inbox の健全状態（Debt < Baseline × 50%）を **継続している時間** に応じて  
最大 20% のボーナスが加算されます。

```
StreakBonus = 1 + min(0.2, healthyHours × 0.005)
```

| 継続時間 | ボーナス |
|:--------|-------:|
| 0時間    |  +0%   |
| 20時間   |  +10%  |
| 40時間以上 | +20%（上限）|

---

## Step 5 ： Price（表示価格）

暗号資産の「価格」として表現するための最終値です。

```
Price = MFI × 10 × StreakBonus
```

| MFI  | StreakBonus | Price |
|-----:|------------:|------:|
| 100  | 1.0         | 1,000 |
|  91  | 1.2         | 1,092 |
|  60  | 1.0         |   600 |
|  36  | 1.0         |   360 |

---

## Step 6 ： Volume（出来高）

「処理した未読負債の量」を Volume として記録します。

```
Volume = max(0, 前回スナップショットの Debt − 現在の Debt)
```

Debt が増えたとき（未読が溜まったとき）は Volume = 0。  
Debt を減らしたとき（メールを処理したとき）のみ Volume に加算されます。

---

## データ収集

スナップショットは **5分に1回** 記録されます（ダッシュボード表示時 & 定期同期時）。

チャートは記録されたスナップショットを **1時間ごとに OHLC 集計** したローソク足で表示します。

| OHLC  | 内容                |
|:------|:-------------------|
| Open  | その時間帯最初の Price |
| High  | その時間帯最大の Price |
| Low   | その時間帯最小の Price |
| Close | その時間帯最後の Price |

---

## Mattermost DM 通知

MFI が **50 を下回った**タイミングで、Mattermost の Bot から DM が届きます。  
同一ユーザーへの通知は **12時間に1回** に制限されています（連投防止）。

通知条件を変更したい場合は `src/lib/mfi.ts` の `MFI_ALERT_THRESHOLD` を変更してください。

---

## まとめ：スコアが動くタイミング

| 行動                         | MFI の動き |
|:----------------------------|:---------|
| 今届いたメールを即読みした        | ほぼ変化なし |
| 昨日のメールを既読にした          | 小幅上昇    |
| 3日以上前のメールを処理した        | 大幅上昇    |
| 古い未読を放置し続けた            | 時間経過で下落 |
| 健全状態を40時間以上維持した       | +20% ボーナス |

> MFI の目的は「未読ゼロを強制すること」ではありません。  
> **古い未読負債を継続的に減らすこと**でスコアが上がります。
